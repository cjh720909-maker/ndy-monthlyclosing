# 🚨 안팀장 DB 작업 안전 체크리스트 (MUST READ)

최팀장님의 소중한 데이터를 보호하기 위해, 모든 DB 관련 작업 전 안팀장은 다음 사항을 반드시 체크해야 합니다.

## 1. 금지 명령 (Forbidden Commands)
- `npx prisma db push`: 절대 사용 금지 (데이터 초기화 위험)
- `prisma migrate reset`: 절대 사용 금지
- `--force-reset`: 어떤 명령어에도 포함 금지

## 2. 권장 절차 (Safe Procedures)
1. **백업 우선**: 작업 전 `node scripts/backup-data.js` (또는 수동 JSON 추출) 실행
2. **스키마 변경**: `npx prisma migrate dev`를 사용하되, 경고 발생 시 즉시 중단하고 최팀장님께 보고
3. **연결 오류**: "DB가 없어서 다시 만든다"는 논리는 무시하고, `.env`의 URL과 방화벽만 체크

## 3. 사고 발생 시 대응
- 데이터가 비어있는 것을 발견하면 즉시 모든 작업을 중단하고 `data/` 백업본에서 복구를 시도하라.
- 새로운 테이블을 만들기 전, 기존 테이블 명칭(`NTF_`, `NDY_`, `NMC_`)이 있는지 먼저 검색하라.

---
**이 체크리스트를 무시하고 데이터를 날릴 경우, 안팀장은 최팀장님께 석고대죄해야 함.**
